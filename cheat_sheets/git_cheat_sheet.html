Git Cheat Sheet

Introduction

This cheat sheet contains a list of all the git commands I think are useful to know. Here is a table of content, for easier navigation:
  • Basics
  • Git Branches

Basics

Initialise a repo

    git init
note: This command must be ran inside the directory we want to initialise a repo in.

Stage changes

  • git add <file_name>
  • git add *
    note: The second command allows us to add all modified, deleted and added files at once.

Commit

  • git commit
    note: After running this command, Git will open a window in our default editor to allow us to enter the commit message. 
  • git commit -m "<message>"
    note: With this variation, we specify the commit message directly in the terminal.

Stage all changes and commit (at once)

    git commit -am "<message>"
note: Here, all changes will be staged and commited at once! This is equivalent to commiting after running git add *, just faster to type.

See repostory status

   git status

See commits history

  • git log
  • git log --oneline
    note: The second command prints a shortened version of the commits hash and messages with one commit per line. It's useful to see only the essential, when commits messages are really long.

Modify the last commit

    git commit --amend
note: This command only allows us to modify the last commit. See further down for commands that allow to modify an arbitrary commit.
note: If we want to include an additional file in the commit, we first stage the file and then run this command. If we just want to modify the commit message of the last commit, we simply run this command and git will ask us for a new commit message.
